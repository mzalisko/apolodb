# Бізнес-правила (для SDD)

## Резолв показу елемента для країни C
1. Якщо елемент прихований вручну → не показувати.
2. Гео: ALL → показувати; ONLY → показувати лише якщо C ∈ countries; EXCEPT → показувати якщо C ∉ countries.

## Телефон: який номер віддається
- Беремо перший entry ланцюга з available=true (active, інакше reserve1, далі reserve2).
- Якщо доступний active → «показується основний»; якщо резерв → «показується резерв N (основний недоступний)».
- Якщо жоден недоступний → значення за замовчуванням / слот прихований.
- Показ форматується маскою (останні N цифр канонічного у шаблон).

## Ціна: який варіант віддається для країни C
- Перший variant, чиє гео збігається з C (ONLY зі списком, що містить C; EXCEPT без C), інакше — variant ALL (за замовчуванням).
- Якщо збігу немає і немає ALL → ціни для країни немає.
- Число обов'язкове; валюта опційна (порожня → показується лише число). Масова зміна ціни — тільки встановлення значення (без ±%/±сума).

## Мітки гео (UI, єдиний стандарт)
- ALL — нейтральний тег; ONLY → тег «ТІЛЬКИ <коди>» (акцент); EXCEPT → тег «КРІМ <коди>» (червоний, коди закреслені в пілюлях).

## Масові зміни (крок-майстер)
- Область: пошук + фільтр групи, «сайт разом із піддоменами», черга обраних.
- Режими значення: однакове / індивідуально по сайтах / знайти→замінити.
- Ціни: встановити значення + вибір області варіантів (усі / лише ALL / варіант країни) або створити гео-варіант.
- Перед застосуванням — прев'ю «сайт · елементів · було → стане»; застосування пише один запис у журнал.

## Доступ
- admin — повний; manager — за матрицею (per-site 0/1/2).
- Призупинений користувач не має доступу (комірки заблоковані).
- «Видати доступ»: рівень + область (групи цілком та/або окремі сайти через пошук).


## Статус «бурштин» = резерв (однозначно)
«Бурштин / резерв» — це **виключно UI-індикатор стану НОМЕРА**: слот телефону зараз віддає резервний номер із ланцюга, бо основний позначений недоступним. Це та сама механіка, що «Ланцюг резервів», а не окрема.

**Резервного вузла/сервера/failover інфраструктури в дизайні НЕМАЄ.** Немає окремого backup-сервера, DNS-перемикання чи інстансу, на якому «працює сайт». Панель і плагін — це керування контактними даними, а не хостинг сайтів.

- Хто перемикає: система автоматично бере наступний доступний запис ланцюга (active → reserve1 → reserve2).
- Чим виявляється недоступність: зовнішня подія/команда «номер не працює» (через синхронізацію/API), не пінг сервера.
- Повернення: коли основний знову доступний — віддається основний.
- Отже «сайт на резерві» = у сайту є хоча б один номер, що зараз віддає резерв. Жодного серверного значення.

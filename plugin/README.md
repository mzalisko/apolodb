# Данные сайта — WordPress-плагін (приймач)

Vanilla PHP на WordPress Plugin API, **без Laravel** (Конституція, Принцип I). Нейтральна
ідентичність — жодних згадок централізованої панелі.

Для цієї фічі плагін періодично (~1/хв) надсилає **підписаний heartbeat** статусу підключення на
єдину публічну точку приймання (проксі).

## Встановлення

Скопіювати теку `plugin/` у `wp-content/plugins/data-site/` і активувати в адмінці.

## Налаштування (вкладка «Подключение»)

- **Адрес приёма** — публічний ендпоінт проксі (напр. `https://ingest.example`).
- **Идентификатор сайта** (`site-id`) — публічний, видається CRM при реєстрації.
- **Секретный ключ** — секрет підпису; показується в CRM ОДИН РАЗ, вставляється сюди й далі не
  показується повторно.

Альтернативно — через `wp-config.php` (секрет поза БД):

```php
define('SD_SIGNING_SECRET', '...');
define('SD_ENDPOINT_URL', 'https://ingest.example');
```

## Надійність відправки (важливо)

WP-Cron спрацьовує лише за відвідувань. Для стабільної відправки раз на хвилину на
низькотрафічних сайтах вимкніть псевдо-крон і викликайте `wp-cron.php` системним cron:

```php
// wp-config.php
define('DISABLE_WP_CRON', true);
```

```
# crontab
* * * * * wget -q -O - "https://ВАШ-САЙТ/wp-cron.php?doing_wp_cron" >/dev/null 2>&1
```

## Підпис (contract §1)

`HMAC-SHA256` над канонічним рядком; заголовки `X-DB-Site-Id/Timestamp/Nonce/Signature/Sig-Version`.
Канонічний рядок і бекенд ПОВИННІ збігатися байт-у-байт (`includes/class-sd-signer.php` ↔
`crm/app/Support/CanonicalRequest.php`).
